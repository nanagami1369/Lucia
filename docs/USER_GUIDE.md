# Lucia ユーザーガイド

Lucia は、Windows RDS ホストの管理ダッシュボードです。ブラウザから RDP セッションの確認・ログオフや、サーバーの電源操作を行えます。

![Luciaダッシュボード](https://raw.githubusercontent.com/nanagami1369/Lucia/main/docs/screenshot.png)

## ⚠️ 安全上の注意

**Lucia は LAN 内での使用を前提として設計されています。**

認証・認可・通信暗号化の機能を持ちません。ダッシュボードにアクセスした人は誰でも、サーバーのシャットダウンや他ユーザーのセッションログオフを含むすべての操作を実行できます。

**インターネットおよび社外ネットワークへの公開は行わないでください。**

## 動作環境

| 項目 | 要件 |
|---|---|
| OS | Windows 10 / 11（64 ビット） |
| 実行権限 | 管理者権限 |

## ダウンロード

以下のページから `Lucia.Installer.exe` をダウンロードします。

```
https://github.com/nanagami1369/Lucia/releases/latest
```

### ダウンロードしたファイルの検証

同じページに `Lucia.Installer.exe.sha256` が公開されています。ダウンロードしたファイルが正しいかどうかを確認するには、両方のファイルを同じフォルダーに置き、PowerShell で以下を実行します。

```powershell
$hash = (Get-FileHash -Algorithm SHA256 .\Lucia.Installer.exe).Hash.ToLower()
$expected = (Get-Content .\Lucia.Installer.exe.sha256).Split(" ")[0]
if ($hash -eq $expected) { "OK" } else { "NG" }
```

`OK` と表示されれば問題ありません。`NG` と表示された場合はファイルが破損しているため、再度ダウンロードしてください。

## インストール

コマンドプロンプトまたは PowerShell を**管理者として**起動し、以下を実行します。

```
Lucia.Installer.exe install
```

インストール内容の確認メッセージが表示されます。

```
=== Lucia インストール ===
  インストール先: C:\Program Files\Lucia
  ポート        : 6100
  許可サブネット: 192.168.0.0/16
インストールを開始します。続行しますか？ [y/N]
```

`y` を入力すると、以下の処理が順に実行されます。

1. 既存サービスの停止・削除（初回は自動的にスキップされます）
2. ファイルの展開
3. イベントログソースの登録
4. Windows Service の登録と起動
5. ファイアウォールルールの設定

すべて完了すると `=== インストール完了 ===` と表示され、サービスが起動した状態になります。

### ポート番号やインストール先を変更する場合

デフォルトの設定を変えたい場合は、オプションを指定して実行します。

```
Lucia.Installer.exe install -p 8080 -i "D:\Lucia" -s 10.0.0.0/8
```

| オプション | デフォルト値 | 内容 |
|---|---|---|
| `-p` | `6100` | ダッシュボードのポート番号 |
| `-i` | `C:\Program Files\Lucia` | インストール先ディレクトリ |
| `-s` | `192.168.0.0/16` | ダッシュボードへの接続を許可する IP の範囲（CIDR 表記） |

確認メッセージを省略して即座にインストールする場合は `--silent` を付けます。

```
Lucia.Installer.exe install --silent
```

## ダッシュボードを開く

ブラウザで以下の URL を開きます。ポート番号を変更した場合は `6100` の部分を置き換えてください。

```
http://<サーバーの IP アドレスまたはホスト名>:6100
```

ダッシュボードが表示されます。画面は「電源管理」と「セッション管理」の 2 つのエリアで構成されています。

### ヒント：ダッシュボードが表示されない場合

- `services.msc` を開き、`Lucia Session Monitor` が「実行中」になっているか確認してください。停止している場合は右クリックして「開始」を選択します。
- 別の PC からアクセスしている場合は、その PC の IP アドレスがインストール時に指定した許可範囲（`-s`）に含まれているか確認してください。

## 電源管理

画面上部に 3 つのボタンが並んでいます。

**再起動** — ボタンを押すと確認ダイアログが表示されます。確定するとサーバーが即時再起動します。

**シャットダウン** — ボタンを押すと確認ダイアログが表示されます。確定するとサーバーが即時シャットダウンします。

**予約シャットダウン（2時間後）** — ボタンを押すと、2 時間後にシャットダウンが予約されます。予約中は画面上部にシャットダウン予定時刻と残り時間が表示され、隣に「キャンセル」ボタンが現れます。予約を取り消すには、この「キャンセル」ボタンを押してください。

## セッション管理

画面下部に、サーバーに接続中の RDP セッションが一覧で表示されます。各セッションには以下の情報が表示されています。

- **ユーザー名** — 接続しているユーザーの名前
- **接続状態** — Active（操作中）、Disconnected（切断済み）、Idle（待機中）のいずれか
- **ログイン** — ログインした時刻
- **アイドル** — 最後の操作からの経過時間

この一覧は 5 秒ごとに自動で更新されます。手動で更新する必要はありません。

### セッションのログオフ

各セッションの下部に「切断」ボタンがあります。ボタンを押すと確認ダイアログが表示され、確定するとそのユーザーのセッションを強制ログオフします。

### RDP サービスの再起動

セッション管理エリアの右上に「RDS再起動」ボタンがあります。これを押すと RDP サービス（TermService）が再起動されます。**接続中の全セッションが切断されるため、注意して使用してください。**

## アップデート

新しいバージョンの `Lucia.Installer.exe` をダウンロードし、インストールと同じ手順で `install` を実行します。既存のサービスは自動的に停止・削除されたうえで、新しいバージョンに置き換わります。

## アンインストール

コマンドプロンプトまたは PowerShell を**管理者として**起動し、以下を実行します。

```
Lucia.Installer.exe uninstall
```

確認メッセージが表示されます。`y` を入力すると、以下が順に削除されます。

1. Windows Service（`LuciaServer`）の停止と削除
2. ファイアウォールルールの削除
3. イベントログソースの削除
4. インストールディレクトリの削除

インストール先を変更していた場合は `-i` で同じパスを指定してください。

```
Lucia.Installer.exe uninstall -i "D:\Lucia"
```

## ログの確認

動作に問題がある場合は、Windows イベントビューアーでログを確認します。

`eventvwr.msc` を開き、**Windows ログ → アプリケーション** を選択します。ソースが `LuciaServer` のエントリに Lucia の動作ログが記録されています。
